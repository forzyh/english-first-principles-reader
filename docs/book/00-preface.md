# 写在前面：把英语重新看成一个系统

传统英语教材往往从名词、动词、时态和从句逐项展开。每一项都能学会，却很难回答一个更根本的问题：这些规则为什么会长成今天这样，它们彼此之间又是什么关系？结果是知识点越积越多，学习者脑海中的结构反而越来越松散。

本书采用另一条路线：把语言看成一种**意义编码协议**。说话者先在头脑中形成对象、事件、关系、时间、立场和交际目的，再把这些多维信息压缩成一条按时间展开的声音或文字；听者则从这条线性信号中重建结构。语法的首要任务，不是制造术语，而是让结构可以被恢复。

> **全书总命题**：英语句法可以近似理解为一棵层级树；句子所表达的现实更像一张语义图；时态、体、情态和语态是加在事件上的元数据；篇章把多棵句法树连接为持续更新的共享模型。

“AST”是一个有力量的类比，但不是等号。编程语言追求唯一解析和确定执行，自然语言则允许省略、歧义、暗示、礼貌和语境推断。因此，本书会始终区分四个层次：

| 层次 | 核心问题 | 典型单位 |
| --- | --- | --- |
| 线性信号 | 实际听见或看见了什么？ | 字符、单词、语序、重音 |
| 句法树 | 哪些词先组合成一个单位？ | 短语、从句、主干、挂接 |
| 语义图 | 现实中有哪些对象和关系？ | 对象、事件、角色、关系边 |
| 语境与语用 | 说话者真正想让我怎样理解？ | 常识、指代、意图、语域 |

这套模型的目标不是把每句话画成复杂的语言学树，而是提供一个稳定的思考框架：遇到新知识时，知道它应该挂到哪里；遇到长句时，知道先找什么；遇到错误时，知道从哪一层调试。

## 本书怎样使用

第一次阅读时，不必追求把每一张图和每一个术语全部记住。先沿着主线读完：

- 先建立“对象—事件—关系”的最小世界模型；
- 再理解英语怎样用主干和短语把它线性化；
- 然后学习冠词、介词、修饰和从句怎样继续扩展树；
- 最后给事件添加时间、视角、现实距离和篇章功能。

第二次阅读时，把自己遇到的真实句子放进对应章节。每次不要问“这是什么规则”，而先问四个问题：

- 这句话创建或恢复了哪些对象？
- 核心事件是什么，动词要求哪些槽位？
- 额外信息挂在名词、事件，还是整个命题上？
- 说话者给事件加了怎样的时间、视角和立场标签？

第三次阅读时，使用第五部分的“解析—生成—调试”算法，把知识转化为操作能力。

## 记号约定

本书使用少量半形式化记号帮助你看清结构：

```text
OPEN(agent=BOY, patient=WINDOW)
```

表示一个 OPEN 事件，施事是男孩，受影响对象是窗户。方括号表示一个可以整体占据槽位的结构：

```text
I know [that he left].
```

箭头表示关系或信息流：

```text
dog_1  <- create: a dog
       <- retrieve: the dog
       <- track: it
```

这些记号不是要求背诵的正式逻辑语言，而是“临时脚手架”。理解稳定后，你应当能够直接在自然句子中看见同样的结构。
