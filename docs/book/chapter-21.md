# 第21章　段落：把多棵树组织成论证

## 21.1 段落是一个局部语义程序

一个有效段落通常执行一个主要交际任务：提出命题、解释机制、提供证据、比较方案、描述过程或回应异议。句子不是按“看起来差不多长”聚在一起，而是共同更新一个中心命题。

可以把说明性段落抽象为：

```text
中心命题
├── 定义或范围
├── 原因 / 机制
├── 证据 / 例子
├── 限制 / 反例
└── 结论 / 过渡
```

## 21.2 主题句不是固定的第一句

主题句的功能是让读者知道本段要更新哪个节点。它可以在开头提出，也可以在例证后归纳，但学术和说明写作常把它放前面以降低解析成本。

示例：

> **The present perfect is best understood as a relation to a current reference point.** It can report a present result, summarize experience, or describe a situation that continues into the present. These uses look different on the surface, but all ask the listener to interpret an earlier event from “now.” This is why a closed past-time expression such as *yesterday* usually selects the simple past instead.

每句都在支持第一句的中心模型，没有突然切换话题。

## 21.3 旧—新链让段落自然向前

一个常见推进模式是：前句末端的新信息，成为后句起点的已知信息。

```text
S1: ... a new monitoring system.
S2: The system collects error logs.
S3: These logs reveal recurring failures.
S4: The failures point to a memory leak.
```

这像指针逐步移动。若连续数句都从完全新的主语出发，读者需要不断重建局部模型，段落会显得跳跃。

## 21.4 平行结构构造比较框架

比较方案时，先建立相同维度：

> The first option is cheaper, easier to deploy, and less secure. The second costs more, requires specialist support, and offers stronger access control.

对应维度清晰，读者可直接对齐。若第一句谈价格，第二句谈团队情绪，第三句突然谈时间，信息图缺少共同坐标。

## 21.5 例子必须回连中心命题

例子不是插入一个有趣故事，而是为抽象节点提供实例。例子后应说明它证明什么：

> Consider the sentence *I saw the man with a telescope*. The final prepositional phrase can describe the instrument of seeing or the man’s possession. **This ambiguity shows that linear adjacency alone cannot determine syntactic attachment.**

最后一句把实例重新连回“层级与挂接”的中心命题。

## 21.6 段落边界是模型重置提示

当以下任一变化发生时，通常考虑新段落：

- 中心对象改变；
- 交际任务改变，如从解释转为评价；
- 时间或场景显著切换；
- 论证进入新理由或回应新异议；
- 当前段落已完成一次完整更新。

段落过长会隐藏结构，过短则会让每个句子都像孤岛。边界应与语义任务对齐。

## 21.7 一个可复用的段落生成器

```text
1. 本段要让读者接受什么中心命题？
2. 读者需要哪些前提或定义？
3. 最有力的机制、证据或例子是什么？
4. 是否有必要说明限制条件？
5. 最后一句怎样回扣并连接下一段？
```

这比“写够五句”更可靠，因为句数是表面结果，语义任务才是组织原则。

### 小练习

围绕命题“冠词是引用管理系统，而不只是名词前的小词”，写一个五到七句段落。至少包含一个例子、一次解释和一个限制条件。
