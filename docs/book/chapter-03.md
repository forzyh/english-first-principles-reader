# 第3章　英语怎样渐进式披露信息

## 3.1 先建立骨架，再追加细节

英语常表现出一种“渐进式披露”倾向：先让听者获得可运作的主干，再把较长、较复杂的信息放到后面。看下面的扩展过程：

```text
The boy opened the window.
The young boy opened the window.
The young boy slowly opened the window.
The young boy slowly opened the window this morning.
The young boy slowly opened the window that faced the garden this morning.
```

每一步都保留了前一步的核心事件，只是在不同节点上增加属性、时间或子树。

这种倾向与实时处理有关。听者听到 The young boy opened... 时，已经建立了一个主语和事件预期；后续信息可以在已有结构上继续挂接。

## 3.2 英语并非“所有修饰都在后面”

“中文修饰在前、英语修饰在后”只能作为粗略直觉，不能当成规则。英语名词前也可以堆叠修饰：

> a carefully designed long-term urban transport policy

真正更稳定的规律是：

- 短、词汇化、易预测的属性常在名词前；
- 长、带内部结构、需要展开的限定常在名词后；
- 重信息和新信息往往靠后；
- 说话者会避免让听者长期悬空等待中心词。

比较：

> the proposal **approved yesterday**

> the **yesterday-approved** proposal

后者在特殊文体中并非绝对不可能，但前者更自然，因为 approved yesterday 自己就是一个带时间成分的结构，放在名词后更易解析。

## 3.3 中心词先行与右分支倾向

很多英语结构以中心词为起点，再向右展开补语：

```text
believe [that the plan will work]
proud [of what the team achieved]
the decision [to postpone the launch]
the report [that you sent yesterday]
```

这使句子常呈现“右分支”形态。对中文母语者而言，最重要的迁移不是强迫自己永远先说中心词，而是学会：**主干足够明确后，不必等所有细节准备完才开口，可以在右侧逐步追加。**

## 3.4 末端重量与末端焦点

英语偏好把较重的成分放在后面，称为 end weight；也偏好把最重要的新信息放在末端，称为 end focus。比较：

> That the project had failed because of a minor configuration error surprised everyone.

> It surprised everyone that the project had failed because of a minor configuration error.

第二句用占位符 it 先填主语位置，把很长的内容从句放到末端。它并非“更有逻辑”，而是更符合实时处理和信息焦点的需要。

### 小练习

把下面信息按英语的渐进披露方式组织为一句自然句子：

- 对象：一份报告；
- 动作：经理读了；
- 报告由新来的分析师撰写；
- 经理在会议前仔细地读；
- 报告讨论客户流失问题。

先写最小主干，再逐层扩展。
