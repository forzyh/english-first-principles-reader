# 第1章　语言为什么需要语法

## 1.1 多维意义必须经过线性通道

现实中的一个场景同时包含许多维度：谁参与、发生了什么、对象之间有什么关系、事件在什么时候发生、说话者是否确定、听者是否知道相关对象。可语言在传输时却通常只能一个词接一个词地展开。

假设你看到这样的场景：一个年轻男孩今天早晨缓慢地打开了面向花园的那扇窗。头脑中的信息并不是按单词顺序储存的，而更像一组并行关系：

```text
BOY(x)
YOUNG(x)
WINDOW(y)
FACE(y, GARDEN)
OPEN(e)
agent(e)=x
patient(e)=y
manner(e)=SLOW
before(e, NOW)
time(e)=THIS_MORNING
```

英语必须把它压缩成一条线：

> The young boy slowly opened the window that faced the garden this morning.

语法就是压缩协议的一部分。它告诉听者：the young boy 应先组合成一个名词短语，slowly 修饰 opened，that faced the garden 挂在 window 上，this morning 给主事件提供时间背景。没有层级结构，词序只是一串符号；有了层级结构，听者才能恢复原来的关系。

## 1.2 语法的本质是“降低恢复成本”

同样的信息可以用许多方式表达，但并不是所有排列都同样容易恢复。比较：

> The boy opened the window slowly.

> Slowly, the window was opened by the boy.

两句都可以指向相似事件，但信息起点、焦点和风格不同。第一句从施事 the boy 出发，第二句从受影响对象 the window 出发。语法不仅传递“发生了什么”，还安排“从哪里进入这个场景”。

因此，语法至少解决五类恢复问题：

| 恢复问题 | 英语常用机制 |
| --- | --- |
| 哪些词属于同一单位？ | 词序、短语结构、重音 |
| 谁是事件的参与者？ | 主语、宾语、介词、格残留 |
| 这个对象是否可识别？ | 冠词、限定词、代词 |
| 事件处于何时、怎样展开？ | 时态、体、时间状语 |
| 说话者如何承诺这个命题？ | 情态、否定、疑问、语气 |

## 1.3 “规则”是稳定策略，不是宇宙定律

很多语法规则其实是长期交流中形成的稳定编码策略。例如，英语通常把限定词放在名词短语前部，把关系从句放在名词后部，把限定动词放在主语之后。它们使听者能够尽早预测后续结构。

学习时应区分三种东西：

- **系统约束**：例如标准英语的限定陈述句通常要求显式主语；
- **高频倾向**：例如较短、较熟悉的信息常靠前，较长、较新的信息常靠后；
- **词汇约定**：例如 depend on、interested in 需要作为构式和搭配掌握。

把所有现象都叫“规则”会造成两个问题：一是把倾向误当绝对，二是把词汇搭配误以为能从空间图式机械推导。第一性原理不是拒绝记忆，而是先判断究竟需要理解什么、记忆什么。

## 1.4 本章操作模型

遇到任何语法现象，先按以下顺序提问：

```text
意义中有什么区别？
        ↓
英语需要让听者恢复什么？
        ↓
它用了哪种可观察信号？
        ↓
这个信号是系统约束、使用倾向，还是词汇约定？
```

### 小练习

解释为什么下面两句虽然描述相近事件，却不完全等价：

- A solution appeared.
- There appeared to be a solution.

提示：不要只比较词义，要比较信息起点、存在结构和说话者的呈现方式。
